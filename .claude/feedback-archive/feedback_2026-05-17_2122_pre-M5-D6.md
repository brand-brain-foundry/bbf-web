## §7 — STOP 6: T1 — useSurface DOM traversal estrategia React 19

### §7.1 Strategic decisión doctrinal

**Strategic decisión:** En React 19 + RSC (Server Components), el DOM traversal por ref es **anti-pattern**. Server Components NO tienen refs en cliente.

**Estrategia canon BBF:**
- Surface se propaga via **SurfaceContext** (preferido, funciona en RSC + Client)
- **NO** se intenta DOM traversal por ref
- Componentes que necesitan surface override: prop explícita `surface={...}`
- Componentes que heredan: useContext(SurfaceContext)

### §7.2 Documentar decisión

Agregar comentario canon en `useSurface.ts` (ya incluido en STOP 5 patch).

### §7.3 TD resolución

```

  Estrategia DOM traversal descartada para React 19 + RSC.
  Surface flows ÚNICAMENTE via SurfaceContext.
  Decisión doctrinal D-97 NUEVA: surface flow context-only.
```

**Strategic registra D-BBF-WEB-97 NUEVA:**
